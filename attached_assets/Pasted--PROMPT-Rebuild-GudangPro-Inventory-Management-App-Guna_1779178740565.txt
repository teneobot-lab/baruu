## 📋 RINGKASAN APLIKASI

Bangun aplikasi **GudangPro** — sistem manajemen inventaris gudang berbasis web, full-stack, dengan tampilan mirip software akuntansi Accurate. Aplikasi ini digunakan oleh staf gudang untuk mencatat transaksi barang masuk/keluar, memantau stok, dan membuat laporan mutasi stok.

**Tech Stack:**
- **Frontend:** React 18 + TypeScript + Vite + Tailwind CSS
- **Backend:** Node.js + Express.js + MySQL (mysql2/promise)
- **Library pendukung:** Lucide React (icons), Recharts (grafik), Fuse.js (fuzzy search), jsPDF + jspdf-autotable (export PDF), ExcelJS (export Excel), Decimal.js (presisi angka), UUID

## 📦 FITUR INVENTORY (Stok Barang)

Halaman tabel barang dengan kolom: Kode, Nama, Kategori, Stok (per gudang), Satuan, Min. Stok, Status.

**Master Item fields:**
```typescript
interface Item {
  id: string;        // UUID
  code: string;      // Kode unik barang
  name: string;
  category: string;
  baseUnit: string;  // Satuan dasar (misal: PCS, KG)
  conversions: UnitConversion[]; // Konversi satuan (Lusin = 12 PCS)
  minStock: number;  // Batas minimum stok
  isActive: boolean;
}
interface UnitConversion {
  name: string;      // Nama satuan (Lusin, Krat, Dus)
  ratio: number;     // Angka konversi
  operator: '*'|'/'; // * = kali, / = bagi
}
```

Fitur di halaman inventaris:
- Filter by kategori, gudang, status aktif/tidak aktif
- Cari by nama/kode
- Tambah/Edit/Hapus (modal form)
- Import dari Excel (template bisa didownload)
- Export ke Excel
- Klik baris → buka `StockCardModal` (kartu stok detail per gudang + riwayat mutasi singkat)
- Tandai item tidak aktif (soft delete)

## 💸 FITUR TRANSAKSI

### TransactionForm
Form transaksi untuk tipe `IN` (Penerimaan) dan `OUT` (Pengiriman):

**Header transaksi:**
- Tanggal, Nomor Referensi (auto-generate, bisa diedit)
- Nomor DO (Delivery Order), opsional
- Gudang Asal (wajib)
- Mitra (Supplier/Customer, dari tabel partners)
- Catatan global

**Line item:**
- Nama barang (autocomplete dari master item)
- Satuan (dropdown dari conversions barang + baseUnit)
- Jumlah
- Catatan per baris
- Qty otomatis dikonversi ke base unit saat menyimpan

**Validasi:**
- Stok tidak boleh minus untuk transaksi OUT
- Nomor referensi harus unik
- Minimal 1 baris item

**Backend logic (`transaction.service.js`):**
- Bungkus dalam DB transaction (BEGIN / COMMIT / ROLLBACK)
- Hitung `base_qty = qty * ratio` (jika operator `*`) atau `qty / ratio` (jika `/`)
- Update tabel `stock` (INSERT ON DUPLICATE KEY UPDATE)
- Untuk UPDATE transaksi: revert stok lama dulu, baru proses ulang
- Untuk DELETE: revert stok ke kondisi sebelum transaksi

## ❌ FITUR REJECT

Modul pencatatan barang reject dari outlet/cabang.

**RejectBatch** = satu sesi input reject dari satu outlet:
```typescript
interface RejectBatch {
  id: string;
  date: string;       // YYYY-MM-DD
  outlet: string;     // Nama outlet/cabang
  createdAt: number;
  items: RejectItem[];
}
interface RejectItem {
  itemId: string;
  sku: string;
  name: string;
  qty: number;
  unit: string;
  baseQty: number;    // Selalu dalam satuan dasar
  reason: string;     // Alasan reject (rusak, kadaluarsa, dll)
  inputQty?: number;  // Qty asli yang diketik user
  inputUnit?: string; // Satuan yang dipilih user
}
```

Fitur:
- Tambah/edit/hapus batch
- Master item reject terpisah dari inventaris utama
- Master outlet (nama-nama cabang)
- Export ke Excel per batch atau rekap periode
- Sinkronisasi ke Google Sheets (via Apps Script URL)

## 🗃️ DATABASE SCHEMA (MySQL)

```sql
-- Users & Auth
CREATE TABLE users (
  id CHAR(36) PRIMARY KEY,
  username VARCHAR(50) NOT NULL UNIQUE,
  password_hash VARCHAR(255) NOT NULL,
  full_name VARCHAR(100) NOT NULL,
  role VARCHAR(20) DEFAULT 'STAFF',  -- ADMIN, MANAGER, STAFF
  status ENUM('ACTIVE','INACTIVE') DEFAULT 'ACTIVE',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Master Data
CREATE TABLE warehouses (
  id CHAR(36) PRIMARY KEY,
  name VARCHAR(100) NOT NULL,
  location TEXT,
  pic VARCHAR(100),
  phone VARCHAR(20),
  is_active BOOLEAN DEFAULT TRUE
);

CREATE TABLE items (
  id CHAR(36) PRIMARY KEY,
  code VARCHAR(50) NOT NULL UNIQUE,
  name VARCHAR(150) NOT NULL,
  category VARCHAR(50),
  base_unit VARCHAR(20) NOT NULL,
  min_stock INT DEFAULT 0,
  is_active BOOLEAN DEFAULT TRUE,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE item_units (
  id INT AUTO_INCREMENT PRIMARY KEY,
  item_id CHAR(36) NOT NULL,
  unit_name VARCHAR(20) NOT NULL,
  conversion_ratio DECIMAL(10,4) NOT NULL,
  operator ENUM('*','/') DEFAULT '*',
  FOREIGN KEY (item_id) REFERENCES items(id) ON DELETE CASCADE,
  UNIQUE(item_id, unit_name)
);

CREATE TABLE partners (
  id CHAR(36) PRIMARY KEY,
  type ENUM('SUPPLIER','CUSTOMER') NOT NULL,
  name VARCHAR(100) NOT NULL,
  phone VARCHAR(20),
  email VARCHAR(100),
  address TEXT,
  npwp VARCHAR(50),
  term_days INT DEFAULT 0
);

-- Stock
CREATE TABLE stock (
  id INT AUTO_INCREMENT PRIMARY KEY,
  warehouse_id CHAR(36) NOT NULL,
  item_id CHAR(36) NOT NULL,
  qty DECIMAL(15,4) DEFAULT 0,  -- Selalu dalam base unit
  last_updated TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  FOREIGN KEY (warehouse_id) REFERENCES warehouses(id),
  FOREIGN KEY (item_id) REFERENCES items(id),
  UNIQUE(warehouse_id, item_id)
);

-- Transactions
CREATE TABLE transactions (
  id CHAR(36) PRIMARY KEY,
  reference_no VARCHAR(50) NOT NULL UNIQUE,
  type ENUM('IN','OUT','TRANSFER','ADJUSTMENT') NOT NULL,
  date DATE NOT NULL,
  source_warehouse_id CHAR(36) NOT NULL,
  target_warehouse_id CHAR(36),
  partner_id CHAR(36),
  partner_name VARCHAR(100),
  delivery_order_no VARCHAR(50),
  notes TEXT,
  created_by CHAR(36),
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE transaction_items (
  id INT AUTO_INCREMENT PRIMARY KEY,
  transaction_id CHAR(36) NOT NULL,
  item_id CHAR(36) NOT NULL,
  qty DECIMAL(15,4) NOT NULL,
  unit VARCHAR(20) NOT NULL,
  conversion_ratio DECIMAL(10,4) NOT NULL,
  base_qty DECIMAL(15,4) NOT NULL,
  note VARCHAR(255),
  FOREIGN KEY (transaction_id) REFERENCES transactions(id) ON DELETE CASCADE,
  FOREIGN KEY (item_id) REFERENCES items(id)
);

-- Reject Module
CREATE TABLE reject_outlets (
  id INT AUTO_INCREMENT PRIMARY KEY,
  name VARCHAR(100) NOT NULL UNIQUE
);

CREATE TABLE reject_items_master (
  id CHAR(36) PRIMARY KEY,
  code VARCHAR(50) NOT NULL UNIQUE,
  name VARCHAR(150) NOT NULL,
  base_unit VARCHAR(20) NOT NULL
);

CREATE TABLE reject_item_units (
  id INT AUTO_INCREMENT PRIMARY KEY,
  item_id CHAR(36) NOT NULL,
  unit_name VARCHAR(20) NOT NULL,
  conversion_ratio DECIMAL(10,4) NOT NULL,
  operator ENUM('*','/') DEFAULT '*',
  FOREIGN KEY (item_id) REFERENCES reject_items_master(id) ON DELETE CASCADE
);

CREATE TABLE reject_batches (
  id CHAR(36) PRIMARY KEY,
  date DATE NOT NULL,
  outlet VARCHAR(100) NOT NULL,
  created_at BIGINT NOT NULL
);

CREATE TABLE reject_batch_items (
  id INT AUTO_INCREMENT PRIMARY KEY,
  batch_id CHAR(36) NOT NULL,
  item_id CHAR(36),
  sku VARCHAR(50),
  name VARCHAR(150),
  qty DECIMAL(15,4),
  unit VARCHAR(20),
  base_qty DECIMAL(15,4),
  reason VARCHAR(255),
  input_qty DECIMAL(15,4),
  input_unit VARCHAR(20),
  FOREIGN KEY (batch_id) REFERENCES reject_batches(id) ON DELETE CASCADE
);

-- Music
CREATE TABLE playlists (
  id CHAR(36) PRIMARY KEY,
  name VARCHAR(100) NOT NULL,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE songs (
  id CHAR(36) PRIMARY KEY,
  playlist_id CHAR(36) NOT NULL,
  title VARCHAR(200) NOT NULL,
  youtube_url VARCHAR(500) NOT NULL,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (playlist_id) REFERENCES playlists(id) ON DELETE CASCADE
);

-- System Config
CREATE TABLE system_settings (
  setting_key VARCHAR(100) PRIMARY KEY,
  setting_value TEXT
);
```

```typescript
export function useTheme() {

    (localStorage.getItem('gp_theme') as any) ?? 'dark'
  );
  const toggle = () => setTheme(t => {
    const next = t === 'dark' ? 'light' : 'dark';
    localStorage.setItem('gp_theme', next);
    return next;
  });
  return { theme, toggle, t: THEME[theme] };
}
```

### `client/src/services/api.client.ts`
```typescript
export const API_URL = ''; // proxy via vite.config.ts
export async function apiCall(endpoint: string, options: RequestInit = {}) {
  const res = await fetch(`${API_URL}${endpoint}`, {
    ...options,
    headers: { 'Content-Type': 'application/json', ...options.headers }
  });
  if (!res.ok) {
    const err = await res.json().catch(() => ({}));
    throw new Error(err.message || `HTTP ${res.status}`);
  }
  return res.json();
}
```
